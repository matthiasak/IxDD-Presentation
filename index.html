debounce()
function debounce(func, wait) {
    var timeout, result;
    return function() {
        var context = this, args = arguments;
        // run func when the timeout triggers
        var later = function() {
            timeout = null;
            func.apply(context, args);
        };
        // reset timeout with latest arguments
        clearTimeout(timeout);
        timeout = setTimeout(later, wait);
    };
}

Multi-touch
  • Typically multi-touch event handlers are used to track one or two touches.
  • Multi-touch event handlers can be used to identify custom gestures (i.e. map rotate).
Multi-touch
  1. A quick search for rotational velocity will give you an algorithm (wiki) to calculate rotation of a single touch around a point,
  2. Calculate the point between the two touches (simple midpoint algorithm)
  3. Then calculate rotational velocity of both touches around this center, normalize the results, and then draw the updates to the screen.
Multi-touch events
  • touchstart - when starting touch
  • touchmove - when moving while touching
  • touchend - when releasing finger after a completed touch motion
  • touchcancel - when the touch sequence is cancelled (panning, OS-level events, notifications)
Multi-touch event properties
  • event.touches - all touches for a single or multi-touch event
  • event.targetTouches - get all touches for the target element
  • event.changedTouches - get all changed touches for this event

The secret to building large apps is never build large apps. Break your applications into small pieces. Then, assemble those testable, bite-sized pieces into your big application.
In otherwords... don't unleash a fury of new concepts on your users without a fair warning.

When I hear "JUST BANG OUT CODE THAT WORKS" I think of all the apps I don't use anymore because they gradually lost the ability to iterate.
In otherwords...

  • or isn't simple enough for developers to explain the implementation,
  • or the prototypes and code quality weren't given time to mature,
...the idea might need some refinement.

A note on performance
4 things a browser can animate cheaply:
  1. size -> transform: scale(n)
  2. position -> transform: translateX(npx)
  3. rotation -> transform: rotate(ndeg)
  4. opacity -> opacity: 0...1
A note on performance
Animate as much as you can with declarative CSS (:hover rules, pseudo-classes, keyframes). Browsers can better optimize simple CSS animations and transitions. But these obviously only work for "one-shot" transitions.

Controlling the environment
function prepEnvironment() {
    // Disable selecting of text
    document.onselectstart = function(e) {
        return e.srcElement.type && !!e.srcElement.type.match(/(text|textarea|password)/);
    };

    // Disable (or customize?) context menus
    document.oncontextmenu = function() {
        return false
    };

    // Disable dragging of HTML elements
    document.ondragstart = function() {
        return false
    };
}
Customized tweening
function tween(obj, prop, to, duration, callback) {
    var start = parseFloat(obj[prop]),
        change = to - start,
        currentTime = 0,
        increment = 20;

    var animate = function() {
        currentTime += increment;
        var val = Math.easeInOutQuad(currentTime, start, change, duration);
        obj[prop] = val;
        if (currentTime <= duration) {
            requestAnimationFrame(animate);
        } else {
            callback && callback();
        }
    };
    requestAnimationFrame(animate);
}
Customized tweening
//t = current time
//b = start value
//c = change in value
//d = duration
Math.easeInOutQuad = function(t, b, c, d) {
    t /= d / 2;
    if (t < 1) return c / 2 * t * t + b;
    t--;
    return -c / 2 * (t * (t - 2) - 1) + b;
}
